AutoTouchヘルプドキュメント

v3.5.4-1に適用する

AutoTouchは、「録音」と「リプレー」移動設備の「タッチ」と「クリック」操作の「マクロ」ツールに用いられるツールで、簡単に言えば、移動設備の「タクトフェアリー」である。Luaスクリプトの運行によって、人の操作を模擬できる。スクリプトは録音で生成し、または手工で編集することによって、産出できる。それは、同時に一連のエクステンション関数を提供し、複雑な自動化操作を実現させる。例えば、自動的にゲームをして、ゴールドコインを取得し、大量的に編集し操作し、プログラムの自動化テストを行う。

AutoTouchはジェイルブレイクしたアップル設備に運行され、Cydiaによってインストールしてください。AutoTouchは録音タッチ、実体ボタンなどの操作を支持し、かつタッチ、スクリーンショット、色調べ、写真マッチング、応用を始動閉まりというエクステンション関数を提供する。同時に、一つのスクリプトモールを提供し、スクリプトをダウンロードし、購入でき、また自分のスクリプトをモールに販売できる。

使い方

どのようにインストールするか。

  • AutoTouchは黙認してCydia持ちのBigBossに発表され、直接的にCydiaに捜索し、取り付けられる。BigBossが通常的に24時間に一回の更新ができるので、もっと速やかな、新しいのを獲得すれば、後をご覧ください。

  • AutoTouchは同時にhttp://autotouch.net/repoをも提供し、速やかな更新を獲得するように、Cydiaに添加できる。

  • Cydiaを通して取り付けられる時に、自動的にプログラムの依頼したActivatorとAppListプラグインをインストールできる。Activatorはジェスチャーの操作を獲得し、設置し、AppListはアプリケーションのインストールしたリストを獲得する。この二つのプラグインをアンインストールしないでください。そうでなければ、AutoTouchも一緒にアンインストールされる。

どのように録音するか。

  • いかなる録音した操作のインターフェースに、ボリュームの減少ボタン（または設置したほかのActivator制御動作）を長押し、コントロールパネルがポップアップされ、コントロールパネルに一つの録音ボタンと一つのスクリプトリストが含まれる。
  • コントロールパネルの赤「録音」ボタンをクリックし、振動で提示させ、かつ録音しはじめる。
  • 次に、録音したい操作をしてください。
  • 録音をやめようとする時に、ボリュームの減少ボタン（または前述のほかの制御動作）を長押し、振動でき、かつポップアップウィンドウが出て、「録音終了」と提示する。
  • 次に、AutoTouchにさっきの録音スクリプトを調べ、管理或いは放送できる。スクリプトが設立時間を黙認の名称として、もっと友好的名称を修正できる。

どのように放送するか。

  • 放送しようとするインターフェースにボリュームの減少ボタン（または設定したほかの制御動作）を長押し、コントロールパネルを調節する。
  • コントロールパネルに放送しようとするスクリプトをクリックし、選択してください。
  • 基本的に、次に（このスクリプトに直接的に放送させるに設置するかぎりに）放送フレームがポップアップされ、循環放送の回数、間隔と速度を聞くことにさせる。
  • 「速やかに運行する」をクリックした後、さっきの設置によって、ただちに放送し、この時に、振動があり、始まりを提示できる。放送が終った後に、自動的にやめ、かつフレームがポップアップされ、終了のことを提示する（設定に、フレームの提示を閉められる）。ボリュームボタン（または前述のほかの動作）を長押すことによって、放送を中断させる。
  • もし「後に運行する」をクリックした後、それは「運行に準備する」の状態に入り、この状態において、ボリュームの減少ボタン（または設置中に修正したほかの起動動作）をクリック（長押しではない）することによって、繰り返して何回も起動し、または運行を中断でき、かつ問い合わせフレームがお邪魔になれない。再び長押し（または前述のほかの動作）、「準備運行」の状態から退出する。

放送設定をどのようにプリセットするか。

  • 「運行設置」のインターフェースにスクリプトを「直接的運行」に設定でき、かつそれに黙認の運行設定を設置し、そのようにして、このスクリプトを選択する時に、直接的にこの設定に即して運行でき、運行設定のフレームが出ない。
  • 「運行設置」のインターフェースに、スクリプトに対して、Activatorジェスチャーのタッチ、運行を設定でき、そのようにして、指定されたジェスチャーが出る時、スクリプトが運行しはじめる。

どのようにスクリーンショットするか。

  • ボリュームの減少ボタンを長押し、コントロールパネルを調節し、コントロールパネルの青「スクリーンショット」ボタンをクリックし、現在のスクリーンをキャプチャでき、かつスクリーンショットの時間をドキュメントの名称として、スクリプトのリストに保存される。
  • スクリーンショットの保存がBMP形式の写真ドキュメントである。AutoTouchはこちらのスクリーンショットを利用して、色の調べ、写真のマッチングなどの操作が需要される。

どのようにスクリプトを編集するか。

  • スクリプトリスト右上の「＋」ボタンをクリックし、「スクリプト新規作成」を選択し、スクリプトの新規作成ウィンドウを明けられる。
  • スクリプト編集インターフェースにコードを編集する。
  • それに、「保存」ボタンをクリックし、名称を入力し、保存する。

スクリプトを編集する時に、「関数助手」をどのように使用するか。

  • スクリプト編集インターフェースのボタン区域の上部に二つのボタンがあり、「エクステンション関数」と「常用文法」で、それを利用して、エクステンション関数とLua常用文法を便利に挿入できる。
  • 「エクステンション関数」ボタンをクリックし、関数選択リストが出て、リストの関数をクリックし、エクステンション関数を直接的にスクリプトに挿入でき、次の編集に提供する。
  • 前の「エクステンション関数」リストに、一部分の関数の右に「補助」ボタンがあり、便利なパラメーターの取得ツールを開ける。こちらの「補助」ツールが主に以下のタイプがある。スクリーン座標の取得、色の取り、応用標識(Identifier)の取得、実体ボタンの選択などがある。
  • 「スクリーン座標取得」ツールに、もうスクリプトのリストに保存されたBMP形式のスクリーンショットを開け、指で写真を画素点の程度まで拡大させ、画素点をクリックし、該画素点がスクリーンにある座標を獲得でき、クリックされた画素点が赤になり、選択されたと表記できる。
  • 同じく、「色拾取」ツールに、写真を開け、画素点が見える程度まで拡大させ、画素点をクリックし、画素の色値を取得できる。
  • 「応用標識選択」がappRun/appKill/appState/appIsActiveの時に、目標応用を便利的に選択できる。
  • 「実体ボタン」がkeyDown/keyUpに用いられる時に、目標ボタンを選択する。
  • 「常用文法」をクリックし、Lua言語の常用文法を速やかに挿入できる。

コンピューターにスクリプトをどのように編集し、管理するか。

  • 設定インターフェースにWeb Serverを始動でき、それに、コンピューターのブラウザを通して、提示のURLに訪問し、それに、その中にスクリプトを編集する。
  • 設定インターフェースにWebDAV Serverも始動でき、それにコンピューターにWebDAVクライアント端末ソフトウェアを通して、提示のサイトに接続し、それに、その中にスクリプトを編集する。

スクリプトをどのように暗号化にさせるか。

  • AutoTouchにスクリプトをクリックし、「暗号化」を選択する。
  • 暗号化のパスワードを入力し、パスワードが要らなく、空になるだけいい。
  • 確定をクリックすれば、暗号化を完成でき、かつ同名、.lua.e終わりの暗号化ドキュメントを生成する。
  • 暗号化のスクリプトを選択し、放送し、パスワードがあるのは、提示によって、パスワードを入力してもいい。

スクリプトをモールに発行するか。

  • 自分の素晴らしいスクリプトをショップに発行し、他人にシェアーし、或いは販売する。
  • 商店にスクリプトを販売すれば、スクリプトを暗号化にさせ、かつパスワードを設定する。このようにして、あなたのスクリプトを買うユーザーがいれば、事前に留保した連絡先を通して、あなたに連絡し、スクリプトを購入する。
  • もし、スクリプトだけシェアーすれば、直接的に.luaドキュメントをアップロードできる。もし他人に内容を見たくないと、パスワードのない暗号化を行うことができる。

モールからどのようにスクリプトをダウンロードし、購入するか。

  • 直接的にショップからすべてのスクリプトをダウンロードできる。
  • あるスクリプトはパスワードがあり、作者と連絡して、パスワードを購入する必要がある。
  • スクリプトをダウンロードした後、あなたのスクリプトリストに保存でき、ほかのスクリプトのように、直接的に使用できる。

授権をどのように購入するか。

  • 「設定」インターフェースの「授権」ボタンをクリックし、授権管理インターフエースを開ける。
  • 開けた「AutoTouch授権管理」インターフエースに、「説明」をクリックし、授権協議の細則を調べられる。下記にPayPalまたはタオバオで授権を購入できる。
  • PayPalで購入するのは、支払った後、サーバが自動的に現在の設備（偶然な状況に、成功にアクティブされないが、もし成功になれないと、後ろをご覧ください。あなたのPayPalアカウントメールボックスで授権番号を調べ、また「現在の設備をアクティブする」のことをしてもいい。
  • タオバオで購入するのは、支払った後、システムが「旺旺」を通して授権番号をあなたに送る。授権番号を獲得した後、「私の授権を調べる」ことによって、「授権番号の調べ」で、授権情報を調べた後、それに設備を起動する。
  • 成功に授権した後、改めて「授権」インターフエースに進入する必要があり、プログラムでドキュメントをダウンロードし、授権させる。もしあなたのネットが悪ければ、失敗にダウンロードし、もし失敗すれば、ネットの環境（例えば、WIFIを4Gに、チャイナユニコムを電信に）を換えてください。繰り返して授権のインターフェースに入り、またダウンロードを試す。授権が検証された後、無限な放送時間と全部の機能をもらえる）。
  • それぞれに三つの調べボタンを利用して、あなたの授権記録を調べられ、それに他の設備を起動できる。一旦に新しい設備が同一の授権番号に起動されば、前にこの授権番号を利用して起動させる設備が、起動をキャンセルさせる。
  • システムの制限が一日に順番に起動の操作（毎日に一回起動することではない）だけできる。 もし需要であれば、24時間後に、またほかの設備を起動させる操作ができる。

座標とサイズ体系

AutoTouch v3.5.2前のバージョンに、座標とサイズ体系が「iOSドットマトリクス体系」を利用する。該バージョンから、全部に「画素点体系」に改正され、即ち実際の画素点に即してサイズと座標を判断する。

例えば、古い「iOSドットマトリクス体系」に、iPhone 6 Plusスクリーンサイズが414x736である。該体系に、サイズが「画素サイズから網膜スクリーンを除く縮小拡大値」の結果で、即ちiPhone 6 Plusの実際画素解像度が1242x2208で、縮小拡大の因素が3で、それで、「iOSドットマトリクス体系」に、スクリーンサイズが(1242÷3)x(2208÷3)である。それに対して、v3.5.2後に採用した「画素点体系」に、画素点のサイズによって、簡単に直感的に座標を判断する。タッチ座標、色の調べ、写真のマッチングを設置しても、直接的に画素サイズで判断し、網膜スクリーンの縮小拡大因素を考慮しない。

古いスクリプトにもしヘッドに古いSCREEN_RESOLUTIONの声明があれば、自動的に識別される。例えば、もし前に録音したiPhone 6 Plusのスクリプトがあれば、SCREEN_RESOLUTION=414x736を声明し、サイズ体系の変更に適応させるために、自動的に識別され、処理される。

エクステンション関数

エクステンション関数はLua言語に使用され、人類の携帯電話を操作することを模擬する能力を備えさせる。またスクリーンショット、色の調べ、写真のマッチングなどの機能を提供する。

touchDown(id, x, y)

スクリーンの(x, y)座標に押す。

パラメーター

パラメーター タイプ 説明
id Integer 指で番号を編成し、単一点、複数点のタッチ標識の指にさせる。
x Integer 屏幕x坐标
y Integer 屏幕y坐标

戻り値
なし

例

  1. -- 座標(100, 200)の所に単一指で押す。
  2. touchDown(0, 100, 200); 
  4. -- 三つの指がスクリーンの三箇所に同時に押す。
  5. touchDown(0, 100, 200);
  6. touchDown(1, 200, 300);
  7. touchDown(2, 300, 400);

指を(x, y)座標に移動させる。

パラメーター

パラメーター タイプ 説明

  1. -- 座標（100, 200)に一つの指で押し、それに指を座標(200, 200)に移動する。

  8. touchDown(1, 200, 300);
  9. touchDown(2, 300, 400);
  10. usleep(16000);
  11. touchMove(0, 150, 250);
  12. touchMove(1, 250, 350);
  13. touchMove(2, 350, 450);

findColors(colors, count, region)

すべてのマッチングした「色の指定及びその相対的位置」の長方形区域を調べ、探した長方形区域にマッチングした第一個の色の画素の座標を戻る。該関数にfindImageより高いサッチ効率と適用度があり、例えば、一つのボタンである。findImageのように、全ボタンの写真にマッチングする必要がなく、ボタンのいくつかの色と相対的位置にマッチングしてもいい。使用できるcountパラメーターが限定され、結果の個数にを制限し、0がすべてを調べることを表示し、1が第一個を探すことを表示し、2が前の二個を探すことを表示する。Regionパラメーターがサッチの区域に限定され、{x, y, width, height}のtableタイプで、nilに導入してもいい。

この関数はスクリプトでインターフェースの「エクステンション関数」の「補助」ツールを編集でき、スクリーンショットから色を速やかに選択し、かつ自動的にその対応的位置を取得し、関数パラメーターの位置に挿入する。

下記の矢印が指した画素点の座標が戻り値の座標である。

パラメーター

パラメーター タイプ 説明
colors table ある色及びその相対的位置があり、例えば{{0x00ddff,0,0}, {0x00eeff,10,10}, {0x0000ff,0,20}}で、大きいtableの小さいtableに三つの値が含まれる。一つ目が色値で、第二個と第三個の値は該色が第一色に相対する相対的位置である。その中に、第一の色のtableの相対的位置がいつも(0,0)で、例えば{0x00ddff,0,0}という色で、後の色の位置値が、第一個の色の位置値に相対する。こちらの色と相対的位置関係を利用し、スクリーンから適切な長方形の区域にマッチングできる。
count Integer 多くてもいくつのマッチング画素点を調べられるか。黙認が0で、すべてのマッチング点を調べられることを表示する。もし1であれば、第一個を調べてもいいことを表示し、もし2であれば、前の二つのを調べてもいいことを表示する。探した個数が少なければ少ないほど、スピードが速い。
region table 指定された区域のサッチに限られる。該パラメーターに{x, y, width, height}四つの値が含まれるtableタイプで、四つの値がそれぞれに長方形区域の左上x,y座標と長方形区域の幅と高で、例えば、{100, 100, 200, 200}である。もし区域に限定させたくなければ、nilに導入してもいい。

戻り値

戻り値 タイプ 説明
locations table 調べられた長方形区域にマッチングした第一個の色の画素点の座標である。例えば{{x1, y1}, {x2, y2}, ...}である。

例

  1. -- 例 1:
  2. local result = findColors({{0x00ddff,0,0}, {0x00eeff,10,10}, {0x0000ff,0,20}}, 2, nil);
  3. for i, v in pairs(result) do
  4.     log(string.format("Found rect at: x:%f, y:%f", v[1], v[2]));
  5. end
  7. -- 例 2:
  8. local colors = {{0x00ddff,0,0}, {0x00eeff,10,10}, {0x0000ff,0,20}};
  9. local result = findColors(colors, 0, nil);
  10. for i, v in pairs(result) do
  11.     log(string.format("Found rect at: x:%f, y:%f", v[1], v[2]));
  12. end
  14. -- 例 3:
  15. local colors = {{0x00ddff,0,0}, {0x00eeff,10,10}, {0x0000ff,0,20}};
  16. local region = {100, 50, 200, 200};
  17. local result = findColors(colors, 0, region);
  18. for i, v in pairs(result) do
  19.     log(string.format("Found rect at: x:%f, y:%f", v[1], v[2]));
  20. end

findImage(imagePath, count, fuzzy, ignoreColors, region)

現在のスクリーンサッチマッチングの写真指定区域に、table形式で見つけたすべての区域の左上の座標に戻る。

パラメーター

パラメーター タイプ 説明
imagePath String 写真のルートを調べる必要がある。AutoTouch v3.1.1バージョンから、こちらに完全ルートに導入する必要がなく、AutoTouchドキュメント目録の位置に導入してもいい。即ち「ローカルスクリプト」位置のルート（rootDir関数がドキュメント目録のルートを取得できる）である。例えば“images/spirit.bmp”が“/var/mobile/Library/AutoTouch/Scripts/images/spirit.bmp”と表示し、完全なルートに導入しないでください。
count Integer 多くてもいくつのマッチング画素点を探すか、黙認が0で、すべてのマッチング点を探すことを表示する。もし1であれば、第一個を調べてもいいと表示し、もし2であれば、前の二つのを調べてもいいと表示する。調べられた数が少なければ少ないほど、スピードが速い。
fuzzy float T調べられる混同度で、黙認が1で、すっかりにぴったりすることを表示する。0.5が50%の画素点をマッチングしてもいい。
ignoreColors table 調べに無視の色値が必要で、数値形式で提供し、例えば、{0xffffff, 0x2b2b2b}で、nilに導入してもいい。
region table 指定された区域に制限し、調べられる。該パラメーターが、{x, y, width, height}四つの値を含むtableの類型で、四つの値がそれぞれに長方形区域左上のx,yと長方形区域の幅と高で、例えば、{100, 100, 200, 200}である。もし区域を限定したくなければ、nilに導入してもいい。

戻り値

戻り値 タイプ 説明
locations table マッチング区域左上の座標の数値である。例えば、{{x1, y1}, {x2, y2}, ...}である。

例

  1. -- 例 1:
  2. local result = findImage("images/spirit.bmp", 5, 1, nil, nil);
  3. for i, v in pairs(result) do
  4.     log(string.format("Found rect at: x:%f, y:%f", v[1], v[2]));
  5. end
  7. -- 例 2:
  8. local result = findImage("images/spirit.bmp", 0, 0.6, nil, nil};
  9. for i, v in pairs(result) do
  10.     log(string.format("Found rect at: x:%f, y:%f", v[1], v[2]));
  11. end
  13. -- 例 3:
  14. local result = findImage("images/spirit.bmp", 0, {0xffffff, 0x2b2b2b}, nil};
  15. for i, v in pairs(result) do
  16.     log(string.format("Found rect at: x:%f, y:%f", v[1], v[2]));
  17. end
  19. -- 例 4:
  20. local imagePath = "images/spirit.bmp";
  21. local region = {100, 50, 200, 200};
  22. local ignoreColors = {0xffffff, 0x2b2b2b};
  23. local result = findImage(imagePath, 1, 0.9, ignoreColors, region};
  24. for i, v in pairs(result) do
  25.     local x = v[1], y = v[2];
  27.     log(string.format("Found rect at: x:%f, y:%f", x, y));
  29.     -- 見つけた位置を一回にクリックする。
  30.     tap(x, y);
  31.     usleep(16000);
  32. end

ORIENTATION_TYPE.PORTRAIT_UPSIDE_DOWN 逆スクリーン、Homeボタンが上にあり、実際値が２である。
ORIENTATION_TYPE.LANDSCAPE_LEFT 左に横スクリーン、Homeボタンが右にあり、実際値が３である。
ORIENTATION_TYPE.LANDSCAPE_RIGHT 右に横スクリーン、Homeボタンが左にあり、実際値が４である。
